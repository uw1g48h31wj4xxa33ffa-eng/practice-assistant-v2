# Document Engine

## 目的
行政機関が公開する公式様式（Word等）を原本として利用し、システム上の確認済みデータを自動入力し、書式・表・改ページ等を破壊せず編集可能なファイルとして安全に出力するための自動入力基盤です。

## 対象形式
現時点では **Word (.docx)** のみに対応しています。
（今後Excel対応を予定していますが、現時点では未実装です）

## 使用パッケージ
- `@xmldom/xmldom`: OOXMLの安全なDOM解析・シリアライズ用
- `pizzip`: `.docx`（ZIPアーカイブ）の展開・再圧縮用

## ファイル管理ポリシー
- **原本・出力ファイルはリポジトリ外**へ配置してください。Git管理には含めません。
- **実顧客データ使用禁止**: テストや検証には架空・匿名のデータのみを使用してください。
- **原本上書き禁止**: 読み込んだ原本ファイルは直接変更せず、別名で出力します。

## 実行方法
原本パス（`INPUT_PATH`）や出力先（`OUTPUT_DIR`）は環境変数で指定可能です。指定がない場合はスクリプト内の既定値（`/Users/to/Documents/practice-assistant-input/` 等）が使用されます。

### テストコマンド
```bash
node --test scripts/document-verification/tests/document-engine.test.mjs
```

### 検証コマンド
```bash
node scripts/document-verification/verify-career-up-form1.mjs
```

## 出力制御
- データのステータスが **confirmed**（確認済み）の値のみを出力対象とします。

## 確認のお願い
- シリアライズ処理による微細な差異が生じる可能性があるため、出力されたファイルは最終的に必ず **Microsoft Word** で開き、表示崩れや修復警告が出ないことを確認してください。
